# Schnellstart

Diese Anleitung führt Sie in etwa fünf Minuten durch die Erstellung eines Polymux-Kontos, das Öffnen Ihrer ersten Sitzung und das Ausführen eines Agenten auf einer echten Website.

## 1. Konto erstellen

Registrieren Sie sich unter [polymux.com](/sign-up) mit einer E-Mail-Adresse oder über Google. Sie werden in einen kostenlosen Arbeitsbereich namens _Personal_ versetzt – Sie können später Teammitglieder einladen, aber vorerst befindet sich alles hier.

Der kostenlose Tarif umfasst drei Agenten, zwei gleichzeitige Browser-Sitzungen und 100 MB Arbeitsbereich-Speicher. Das reicht aus, um diesem Leitfaden zu folgen und einiges mehr.

## 2. Starten Sie Ihren ersten Workflow

Drücken Sie im Dashboard in der linken Seitenleiste auf **+ New Workflow**. Ein Entwurfs-Workflow wird geöffnet, mit einem Chatfenster und einem Live-Browser-Viewport daneben.

Geben Sie eine Anfrage wie diese ein:

> Öffne Hacker News und lies mir die Titel der drei wichtigsten Geschichten vor.

Drücken Sie **Send**. Der Agent öffnet einen Browser-Tab, navigiert und streamt die Seite zurück in den Viewport. Alles, was er liest, wird im Chat zusammen mit den Aktionen angezeigt, die er ausgeführt hat.

## 3. Beobachten Sie den Agenten bei der Arbeit

Der Viewport auf der rechten Seite zeigt genau, was der Agent sieht. Sie können:

- **Hovern**, um Elemente hervorzuheben, mit denen der Agent gleich interagieren wird.
- **Übernehmen**, indem Sie _Pause_ klicken – der Agent stoppt, Sie steuern manuell, und ein Klick auf _Resume_ gibt die Kontrolle zurück.
- **Ansichten wechseln** zwischen dem Live-Browser, dem Workflow-Graphen und der Artefakt-Galerie (Screenshots, Dateien, Downloads) über den Umschalter über dem Viewport.

Sitzungen bleiben aktiv, bis Sie sie schließen oder sie beendet werden. Das Schließen des Tabs beendet die Sitzung nicht – sie läuft auf dem Server weiter, und Sie können sie über die Seitenleiste wieder aufnehmen.

## 4. Als Workflow speichern

Ein freier Chat ist eine Sitzung. Um ihn wiederverwendbar zu machen, drücken Sie **Save as workflow** oben im Chat. Polymux erfasst den Agent-Prompt, die verwendeten Werkzeuge und alle referenzierten Tresoreinträge und speichert das Ganze als versionierten Workflow, den Ihr Team erneut ausführen kann.

Workflows können auch geplant werden. Wechseln Sie auf der Workflow-Seite zum Tab **Schedule**, um sie mit einem Cron-Ausdruck auszuführen – täglich, wöchentlich oder in einem individuellen Intervall.

## 5. Wie es weitergeht

Sie kennen jetzt die Grundlagen. Wählen Sie eines:

- **Möchten Sie einen Workflow mit Ihrem Team teilen?** Lesen Sie [Arbeitsbereiche und Mitglieder](/documentation/workspaces).
- **Logins oder API-Schlüssel speichern?** Lesen Sie [Tresor-Grundlagen](/documentation/vault).
- **Hängen Sie fest?** Die [FAQ](/documentation/faq) behandelt die häufigsten Stolpersteine.
- **Bauen Sie etwas auf Polymux auf?** Springen Sie vor zur [Plugin-Übersicht](/documentation/plugin-overview).
