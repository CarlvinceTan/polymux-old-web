# 构建你的第一个插件

本指南将带你将一个现有工作流打包为可安装的插件。它假设你已阅读 [插件概览](/documentation/plugin-overview),并且在你的工作空间中有一个端到端运行干净的工作流。

## 1. 选定一个候选工作流

打开你想发布的工作流。在做其他任何事情之前,先进行一次健康检查:

- **它能从冷启动运行吗?** 在聊天框为空的情况下触发一次新运行。任何依赖先前上下文的内容——之前上传的文件、会话中途创建的保险库条目——都会在安装者从零开始时失败。
- **所有密钥都在保险库中吗?** 如果某个密钥被原样粘贴到提示词中,任何安装该插件的人都会得到它。把它移到保险库中,通过键引用。
- **URL 和 ID 是否可配置?** 把硬编码的组织名称、电子表格 ID 或主机名替换为安装者可填写的工作流输入。

通过此检查清单的工作流就处于良好状态可以打包了。

## 2. 打开 Publish 选项卡

在工作流页面切换到 **Publish** 选项卡。如果你看不到它,说明你的工作空间角色不包含发布权限——请让管理员将你升级到 **Member** 或更高级别。

Publish 选项卡有四个子面板:**Listing**、**Connections**、**Pricing** 和 **Review**。我们将按顺序填写这些内容。

## 3. 列表(Listing)

列表是购物者在市场卡片和详情页上看到的内容。

- **Name。** 1–40 个字符。避免末尾的版本号——版本是单独跟踪的。
- **Short description。** 一句话,少于 120 个字符。它会出现在卡片上。
- **Long description。** 支持 Markdown。说明工作流的功能、它需要什么以及它会产生什么。截图会有帮助。
- **Icon。** 方形 PNG,最低 256×256。可接受 SVG。
- **Category。** 选择最接近的分类。分类用于市场筛选。
- **Tags。** 最多五个自由形式标签。用于搜索。

## 4. 连接器(Connections)

Polymux 会扫描工作流图,并呈现它找到的每一项外部依赖:保险库键、OAuth 提供商、集成 ID。对每一项,声明:

- **它是必需的还是可选的。** 可选连接器允许在未提供时以降级模式运行工作流。
- **显示标签。** 安装者在安装对话框中看到的内容。_"OpenAI API key"_ 比原始保险库键名要好。
- **帮助文本。** 解释安装者应在哪里获取该值的简短提示。如有相关,可链接到提供商文档。

如果连接器是内置的 Google Drive OAuth 提供商,安装者会在安装期间一键授权。对于任何其他提供商,工作流会依赖于市场集成(或安装者粘贴的保险库键);两者都通过同一个安装对话框处理。

完整的字段架构请参见 [插件清单参考](/documentation/plugin-manifest)。

## 5. 定价(Pricing)

三个选项:

- **Free。** 无需付款,无需设置 Stripe。
- **One-time。** 安装者一次付款后,无限期拥有该插件在其工作空间中的使用权。
- **Subscription。** 每月循环计费。安装者可随时取消,期满时插件会被卸载。

对于付费插件,你必须从 **Settings → Payouts** 连接一个 Stripe 账户。Polymux 持有 15% 的平台费,并按月支付余额。最低支付门槛为 50 美元。

## 6. 审核与提交

**Review** 子面板会预览安装者将看到的列表。查看截图、点击体验连接提示,并通读长描述以排查拼写错误。

按 **Submit for review**。插件进入队列,你会在两个工作日内收到邮件了解审核结果。常见的拒绝原因有:

- 列表描述没有写明插件实际能做什么,或它接触哪些数据。
- 必需连接器的标签不够清晰,陌生人无法理解。
- 工作流嵌入了应放入保险库的密钥。

你可以多次编辑列表并重新提交。

## 7. 发布更新

插件上线后,对底层工作流的任何更改都会变成 _新版本_。版本不会自动发布;在 Publish 选项卡中选择 **Promote to public**,使版本可供现有安装者使用。他们可以选择更新或保持在旧版本上。

破坏性更改——例如,要求新增连接器——必须递增主版本号。Polymux 会警告安装者在应用更新之前会被询问新的权限。
