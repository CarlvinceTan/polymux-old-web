# 工作空间与成员

工作空间是团队在 Polymux 中构建的所有内容的共享容器:工作流、保险库条目、文件、账单和成员。你至少属于一个工作空间——你的个人工作空间——并可以根据需要成为任意多个共享工作空间的成员。

## 创建工作空间

在侧边栏中打开工作空间切换器,选择 **+ New workspace**。给它起一个名字,并可选地添加一个头像。新工作空间是空的:没有工作流、没有保险库条目、没有共享文件。创建工作空间的用户是其首位 **所有者(owner)**。

## 角色

Polymux 有四种角色,能力依次递增:

| 角色 | 可运行工作流 | 可编辑工作流 | 可管理成员 | 可管理账单 |
| --- | --- | --- | --- | --- |
| Viewer | 是 | 否 | 否 | 否 |
| Member | 是 | 是 | 否 | 否 |
| Admin | 是 | 是 | 是 | 否 |
| Owner | 是 | 是 | 是 | 是 |

你可以随时从工作空间设置中更改成员的角色。必须始终至少有一名所有者;角色选择器将拒绝降级最后一名所有者。

## 邀请他人

在工作空间设置中,将一组电子邮件地址粘贴到邀请字段中,选择角色,然后点击 **Send invitations**。每位受邀人都会收到一封带链接的邮件,链接七天后过期。待处理的邀请也会出现在同一设置页面;你可以重发或撤销其中任何一项。

如果受邀人在相同邮箱下已有 Polymux 账户,接受邀请会立即将其添加到工作空间。如果他们没有账户,邀请链接会先引导他们完成注册。

## 切换工作空间

侧边栏中的工作空间切换器会显示你所属的每一个工作空间。切换会更改整个上下文:侧边栏会重新加载该工作空间的工作流,保险库和存储选项卡的范围会限定到该空间,你创建的任何新工作流也归该空间所有。任何内容都不会跨工作空间泄漏。

## 工作空间内部共享

默认情况下,每位成员都能看到工作空间中的每一个工作流、保险库条目和文件。没有按资源粒度的 ACL——如果你需要更小的影响范围,请使用更小的工作空间。

但有一个例外:**个人存储(Personal storage)** 中的文件仅对上传者私有。要与队友共享文件或文件夹,请在存储中打开它,并选择 **Share with**。共享可撤销,且会出现在收件人的 _Shared with me_ 选项卡中。

## 移除成员

管理员和所有者可以从工作空间设置中移除成员。被移除的成员会立即失去访问权限;他们正在运行的任何会话不会被中断,但他们将无法再接入。

如果被移除的成员是为某工作流配对了 [浏览器扩展](/documentation/installation#browser-extension) 的唯一一个人,该工作流仍会在托管浏览器上继续运行——扩展配对是按用户而非按工作空间。
