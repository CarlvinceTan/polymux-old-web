# 保险库基础

保险库是用来存放凭据、API 密钥及其他需要供智能体代你执行操作时使用的密钥的地方。智能体从保险库读取的任何内容都会被记录、限定在会话范围内,并且永远不会原样返回给模型——它会被直接注入到页面或工具调用中。

## 保险库中存放什么

保险库条目分为两种:

- **密码(Passwords)** —— 限定到某个域名的用户名 / 密码对。当智能体进入匹配的 URL 时,会将其填入登录表单。
- **钱包条目(Wallet entries)** —— API 密钥、bearer 令牌、OAuth 客户端密钥及其他任意密钥。它们会被注入到工具调用(HTTP 请求头、CLI 参数)中,而不是填入表单。

保险库不存储文件、证书或 SSH 密钥。请使用 [工作空间存储](/documentation/installation) 来存放这些内容。

## 添加条目

在侧边栏中,打开 **Vault → Passwords** 或 **Vault → Wallet**,然后点击 **+ New**。为该条目命名,填写主机(密码条目)或键的结构(钱包条目),以及其值。该值会使用工作空间密钥静态加密;没有人——包括 Polymux 工程师——能够在没有你授权的情况下读回它。

## 智能体如何访问保险库

智能体看不到密钥。当工作流需要某个密钥时,它会向保险库发送一个有类型的请求,例如 _"github.com 的密码"_,Polymux 会将该值注入到下一步操作中,而绝不会将其放入智能体的上下文窗口。模型知道密钥存在以及它用于何处;但它不知道实际的字符。

如果你看到一个会话在登录表单前后短暂暂停又恢复,那次暂停就是保险库正在填充凭据。

## 范围设置

默认情况下,工作空间中具有 **Member** 角色或更高权限的每位成员都可以在工作流中使用任何保险库条目。要对某条目进行限制,请编辑它并设置范围:

- **Workspace** —— 拥有该工作空间角色的任何人都可使用。默认设置。
- **Workflow** —— 仅指定 ID 列表中的工作流可请求。适用于影响重大的密钥。
- **Owner only** —— 仅该条目的创建者本人可使用,包括在调度运行中。

目前还没有审计日志的 UI,但每次保险库读取都会在服务器端被捕获。如果你需要拉取某条目的日志,请 [联系支持](/contact)。

## 轮换密钥

打开条目,点击 **Rotate**,然后粘贴新值。旧值会被擦除——保险库中没有版本历史。引用了旧值的工作流在下一次运行时仍可继续工作;运行中的会话在结束前仍会在内存中持有旧值。

## 删除条目会发生什么

删除是立即生效且不可恢复的。任何在保险库读取被删除条目时暂停的会话都会以 `vault_missing` 失败,并需要新的条目。计划运行也会以同样方式失败。任何失败的运行都会在仪表板中显示通知。
