# 连接器概览

_连接器_ 是工作流连接外部世界的方式:一个经 OAuth 授权的账户、一个保险库存储的 API 密钥、或一个第一方集成。插件在其清单中声明所需的连接器;安装者在安装时授权这些连接器。

本页从安装者的视角出发。如果你正在构建插件,请同时阅读 [构建连接器](/documentation/connections-build)。

## 存在哪些类型的连接器

Polymux 支持三种类型的连接器:

### 保险库连接器

保险库连接器是一种你粘贴到 Polymux 的有类型密钥。安装对话框会告知你插件需要哪个键、它期望的形状(API 密钥、bearer 令牌、basic 认证对),并链接到获取它的位置。

保险库连接器不会离开你的工作空间。插件作者无法读取它们;Polymux 工程师也无法读取它们。唯一接触原始值的,是使用它的智能体动作。

### OAuth 连接器

OAuth 连接器将授权委托给第三方。在安装对话框中点击 **Authorise** 按钮,你会被跳转到提供商的同意界面,返回时令牌会以提供商范围限定的键存储在你的工作空间保险库中。

令牌会自动刷新。如果刷新失败(你撤销了授权、提供商进行了轮换、你的账户被禁用),该连接器会被标记为 _broken_,任何依赖它的插件会暂停,直到你重新授权。

### 集成连接器

集成涵盖了所有不属于内置 OAuth 提供商的内容。大多数集成以社区发布的连接器形式出现在市场中,封装第三方 API;Polymux 的清单描述连接器所需的密钥及其暴露的工具,你以与安装插件相同的方式安装它。

每个集成都有自己的设置流程,带有特定于提供商的字段。配置完成后,集成会作为工作流可调用的工具出现。多个插件可以共享同一个集成,无需重新授权。

## 受支持的提供商

主线内置支持的 OAuth 提供商:

- Google Drive —— 为工作空间存储的核心提供支撑。

其他所有提供商都以可安装的 **集成** 形式存在于市场中。浏览 [Marketplace → Integrations](/integrations/marketplace) 查看目前发布的内容,或自行编写([构建连接器](/documentation/connections-build))。

如果你想要安装的插件依赖于尚未在市场中发布的集成,安装时会按名称列出缺失的依赖,以便你在重试之前获取(或发布)它。

## 安装带连接器的插件

当你在市场列表上点击 **Install** 时,Polymux 会打开一个安装对话框,按顺序引导你完成每个必需的连接器:

1. **阅读描述。** 每个连接器都有标签、帮助说明和必需/可选标记。
2. **授权。** OAuth 连接器显示 _Authorise_ 按钮。保险库连接器显示一个粘贴字段。集成连接器显示相关的提供商表单。
3. **确认。** 摘要页面列出插件将拥有访问权限的所有内容。按 _Install_ 提交。

插件会出现在你的工作空间的插件列表中。除非插件本身有调度,否则首次运行会手动进行;如果有调度,Polymux 会按计划运行它。

## 重新连接已断开的连接器

在插件的设置中,点击任何已断开连接器旁的 **Reconnect**。对话框会引导你完成相同的授权 / 粘贴 / 确认流程。

对于 OAuth 提供商,连接器断开最常见的原因是底层账户被删除或 OAuth 授权在提供商一侧被撤销。Reconnect 会干净地获取新的授权。

## 移除连接器

在插件仍引用某连接器时移除它,会让插件进入降级状态。可选连接器会被静默丢弃;必需连接器则会导致插件在下次运行时失败。

你可以从 **Vault → Connections** 中移除连接器。提供商一侧的授权不会被自动撤销——要执行该操作,请前往提供商的账户设置并在那里撤销。
