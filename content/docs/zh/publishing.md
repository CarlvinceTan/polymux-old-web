# 发布清单

你已经构建了一个插件,填写了列表,Publish 选项卡的 _Review_ 面板看起来也没问题。在按下 **Submit for review** 之前,请运行本清单。大多数被拒绝的原因都来自本页面上的某项。

## 工作流

- [ ] **可以从冷启动运行。** 在聊天框为空的情况下触发一次新运行。工作流必须在第一轮就达到终止状态,无需你干预。
- [ ] **没有硬编码的密钥。** 在提示词中搜索任何看起来像密钥、令牌、密码或邮箱的内容。把每一项都移到保险库中并通过键引用。
- [ ] **没有工作空间特定数据。** 内部主机名、成员邮箱、电子表格 ID、Slack 频道——安装者可能想要更改的所有内容,都必须是工作流输入。
- [ ] **所有工具均为第一方或市场已发布工具。** 从你自己私有工作空间导入工具的插件可以被安装,但永远无法为其他人运行。
- [ ] **在失败时干净地停止。** 触发一个已知的失败路径(错误的保险库条目、离线提供商),并确认工作流呈现一个有用的错误,而不是无休止地转圈。

## 列表

- [ ] **Name** 长度为 1–40 字符,不含版本号。
- [ ] **Short description** 是一句话,少于 120 字符,说明插件 _做什么_ ——而不是 _用什么构建_。
- [ ] **Long description** 说明插件需要什么输入、产生什么输出、接触哪些数据。审核者会仔细阅读。
- [ ] **Icon** 是方形的,在 64×64 时仍能清晰渲染。
- [ ] **Screenshots** 展示插件的使用,而不是营销网站。两到四张是最佳数量。
- [ ] **Category** 是最贴近的分类。_Other_ 仅保留给确实不合适其他分类的插件。
- [ ] **Tags** 准确无误。不要用不相关词条填充——搜索会对标签堆砌的内容降级排名。

## 连接器

- [ ] **每个必需的连接器都有一个让陌生人也能理解的 `label`。** _"OpenAI API key"_ 是好的;_"oai_k"_ 则不行。
- [ ] **每个必需的连接器都有一段 `help` 说明**,除非标签本身已完全自我解释(OAuth 提供商通常如此)。
- [ ] **可选连接器优雅地降级。** 在每个可选连接器缺失的情况下触发一次运行,确认工作流要么跳过该步,要么返回一个合理的错误。
- [ ] **作用域尽量最小。** 请求让工作流能够运行的最窄 OAuth 作用域。审核者会反对过宽的授权。

## 定价(仅付费插件)

- [ ] **Stripe Connect 已设置完成。** `Settings → Payouts` 应显示一个已验证的账户。
- [ ] **货币和金额正确。** 发布之后,列表的货币若需更改,必须联系支持。
- [ ] **退款政策** 已在长描述中提及。平台没有统一政策;由你自行设定。

## 提交之后

- 审核会在两个工作日内完成。你会收到一封包含结果的邮件。
- 如果被拒,邮件会列出需要修改的具体字段。修复后重新提交——多次提交不会有处罚。
- 如果通过,列表会立即上线。市场的 _New_ 选项卡会在前 7 天内对其特别推荐。

## 上线后的版本管理

每次推送对工作流的更改时都应递增版本:

- **Patch**(`1.0.0 → 1.0.1`)—— 错误修复,无行为变化,无新连接器。
- **Minor**(`1.0.0 → 1.1.0`)—— 新增可选行为、可选连接器,或带默认值的新增输入。
- **Major**(`1.0.0 → 2.0.0`)—— 必需新增连接器、移除输入,或行为发生实质性差异。

安装者可以将版本固定在某个范围内。主版本号递增会提示他们在更新前重新授权。

## 取消发布

你可以随时在 Publish 选项卡中将插件从市场中下架。现有安装者会继续运行该插件,直到他们卸载;新的安装者则找不到它。

如果你需要 **紧急下线** 一个插件——例如,你发现它泄露数据——请联系支持。我们可以为所有安装者一次性在服务器端禁用该插件。
