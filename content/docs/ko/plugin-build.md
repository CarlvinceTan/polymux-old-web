# 첫 플러그인 구축

이 안내는 기존 워크플로를 가져와 설치 가능한 플러그인으로 패키지화하는 과정을 다룹니다. [Plugin overview](/documentation/plugin-overview)를 이미 읽었고, 워크스페이스에 처음부터 끝까지 깔끔하게 실행되는 워크플로가 있다고 가정합니다.

## 1. 후보 워크플로 선택

게시하려는 워크플로를 여세요. 다른 작업을 하기 전에 점검부터 합니다.

- **cold start에서 실행되나요?** 빈 채팅에서 새 실행을 트리거하세요. 이전 컨텍스트(이전에 업로드된 파일, 세션 중간에 생성된 vault 항목)에 의존하는 모든 것은 처음부터 시작하는 설치자에게는 실패합니다.
- **모든 비밀 정보가 vault에 있나요?** 비밀 정보가 프롬프트에 그대로 붙여넣어져 있다면, 플러그인을 설치하는 모든 사람이 그것을 갖게 됩니다. vault로 옮기고 키로 참조하세요.
- **URL과 ID는 구성 가능한가요?** 하드코딩된 조직 이름, 시트 ID 또는 호스트네임을 설치자가 입력할 수 있는 워크플로 입력으로 대체하세요.

이 체크리스트를 통과한 워크플로는 패키지화하기에 좋은 상태입니다.

## 2. Publish 탭 열기

워크플로 페이지에서 **Publish** 탭으로 전환하세요. 보이지 않는다면 워크스페이스 역할에 게시 권한이 없는 것입니다. 관리자에게 **Member** 이상으로 업그레이드해 달라고 요청하세요.

Publish 탭에는 네 개의 하위 패널이 있습니다. **Listing**, **Connections**, **Pricing**, **Review**입니다. 순서대로 작성합니다.

## 3. 목록

목록은 마켓플레이스 카드와 상세 페이지에서 쇼핑객이 보는 것입니다.

- **이름.** 1–40자. 끝에 버전 번호는 피하세요. 버전은 별도로 추적됩니다.
- **짧은 설명.** 한 문장, 120자 미만. 카드에 표시됩니다.
- **긴 설명.** 마크다운을 지원합니다. 워크플로가 무엇을 하는지, 무엇이 필요한지, 무엇을 생성하는지 설명하세요. 스크린샷이 도움이 됩니다.
- **아이콘.** 정사각형 PNG, 최소 256×256. SVG도 허용됩니다.
- **카테고리.** 가장 가까운 일치 항목을 선택하세요. 카테고리는 마켓플레이스 필터링을 좌우합니다.
- **태그.** 자유 형식 태그 최대 5개. 검색에 사용됩니다.

## 4. 연결

Polymux는 워크플로 그래프를 스캔하고 발견된 모든 외부 의존성(vault 키, OAuth 제공자, 통합 ID)을 표시합니다. 각각에 대해 다음을 선언하세요.

- **필수인지 선택사항인지.** 선택적 연결은 제공되지 않을 경우 워크플로가 저하된 모드로 실행되도록 합니다.
- **표시 레이블.** 설치자가 설치 대화상자에서 보는 것. _"OpenAI API key"_가 원시 vault 키 이름보다 낫습니다.
- **도움말 텍스트.** 설치자가 값을 어디서 얻어야 하는지 설명하는 짧은 힌트. 관련된 경우 제공자 문서로 연결하세요.

연결이 내장된 Google Drive OAuth 제공자인 경우, 설치자는 설치 중에 한 번의 클릭으로 인증합니다. 다른 제공자의 경우, 워크플로는 마켓플레이스 통합(또는 설치자가 붙여넣는 vault 키)에 의존하며, 둘 다 같은 설치 대화상자에서 처리됩니다.

여기서 캡처되는 전체 스키마는 [Plugin manifest reference](/documentation/plugin-manifest)를 참고하세요.

## 5. 가격

세 가지 옵션입니다.

- **Free.** 결제도, Stripe 설정도 필요 없습니다.
- **One-time.** 설치자가 한 번 결제하고 자신의 워크스페이스에서 무기한 플러그인을 소유합니다.
- **Subscription.** 매월 반복됩니다. 설치자는 언제든지 취소할 수 있고, 플러그인은 기간 종료 시 제거됩니다.

유료 플러그인의 경우 **Settings → Payouts**에서 Stripe 계정을 연결해야 합니다. Polymux는 15% 플랫폼 수수료를 보유하고 나머지를 매월 지급합니다. 최소 지급액 임계값은 $50입니다.

## 6. 검토 및 제출

**Review** 하위 패널은 설치자가 보게 될 목록을 미리 보여줍니다. 스크린샷을 살펴보고, 연결 프롬프트를 클릭해 보고, 긴 설명에서 오타를 확인하세요.

**Submit for review**를 누르세요. 플러그인이 대기열에 들어가고 영업일 기준 2일 이내에 결과가 담긴 이메일을 받습니다. 거절의 일반적인 이유는 다음과 같습니다.

- 목록 설명이 플러그인이 실제로 무엇을 하는지 또는 어떤 데이터를 다루는지 누락함.
- 필수 연결이 낯선 사람이 이해할 만큼 명확하게 레이블이 지정되지 않음.
- 워크플로에 vault에 있어야 할 비밀 정보가 포함됨.

목록을 편집하고 원하는 만큼 다시 제출할 수 있습니다.

## 7. 업데이트 게시

플러그인이 라이브가 되면 기본 워크플로에 대한 모든 변경은 _새 버전_이 됩니다. 버전은 자동으로 게시되지 않습니다. Publish 탭에서 **Promote to public**을 선택하여 기존 설치자가 버전을 사용할 수 있게 하세요. 그들은 업데이트하거나 이전 버전에 머물기를 선택할 수 있습니다.

호환성을 깨는 변경(예: 새 연결 요구)은 메이저 버전을 증가시켜야 합니다. Polymux는 업데이트를 적용하기 전에 설치자에게 새로운 권한을 요청한다고 경고합니다.
