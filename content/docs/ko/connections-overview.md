# 연결 개요

_연결_은 워크플로가 외부 세계에 도달하는 방법입니다. OAuth 인증된 계정, vault에 저장된 API 키, 첫 번째 당사자 통합 등이 있습니다. 플러그인은 매니페스트에서 필요한 연결을 선언하고, 설치자는 설치 시 그러한 연결을 인증합니다.

이 페이지는 설치자 관점에서 작성되었습니다. 플러그인을 구축하는 경우 [Building a connection](/documentation/connections-build)도 함께 읽으세요.

## 어떤 종류의 연결이 있나요

Polymux는 세 가지 연결 종류를 지원합니다.

### Vault 연결

vault 연결은 Polymux에 붙여넣는 타입이 지정된 비밀 정보입니다. 설치자 대화상자는 플러그인이 어떤 키를 필요로 하는지, 어떤 형태(API 키, bearer 토큰, basic auth 쌍)를 기대하는지 알려주고 어디서 얻을 수 있는지에 대한 링크를 제공합니다.

vault 연결은 절대 여러분의 워크스페이스를 떠나지 않습니다. 플러그인 작성자도 읽을 수 없고, Polymux 엔지니어도 읽을 수 없습니다. 원시 값에 닿는 유일한 것은 그것을 사용하는 에이전트 동작입니다.

### OAuth 연결

OAuth 연결은 인증을 제3자에게 위임합니다. 설치 대화상자에서 **Authorise** 버튼을 클릭하면 제공자의 동의 화면으로 이동하고, 돌아오면 토큰이 제공자별 키로 워크스페이스의 vault에 저장됩니다.

토큰은 자동으로 갱신됩니다. 갱신이 실패하면(권한 부여를 취소했거나, 제공자가 교체했거나, 계정이 비활성화된 경우) 연결은 _broken_으로 표시되고 그것에 의존하는 플러그인은 여러분이 다시 인증할 때까지 일시 정지합니다.

### 통합 연결

통합은 내장 OAuth 제공자가 아닌 모든 것을 다룹니다. 대부분 제3자 API를 래핑하는 커뮤니티 게시 커넥터로 마켓플레이스에 존재합니다. Polymux의 매니페스트는 커넥터가 필요로 하는 비밀 정보와 노출하는 도구를 설명하며, 플러그인을 설치하는 것과 같은 방식으로 설치합니다.

각 통합에는 제공자별 필드가 있는 자체 설정 흐름이 있습니다. 한 번 구성되면 통합은 워크플로가 호출할 수 있는 도구로 표시됩니다. 여러 플러그인이 재인증 없이 같은 통합을 공유할 수 있습니다.

## 지원되는 제공자

in-tree로 지원되는 OAuth 제공자.

- Google Drive — 워크스페이스 저장소 기반을 지원합니다.

다른 모든 제공자는 설치 가능한 **통합**으로 마켓플레이스에 존재합니다. 현재 게시된 것을 보려면 [Marketplace → Integrations](/integrations/marketplace)를 탐색하거나 직접 작성하세요([Building a connection](/documentation/connections-build)).

설치하려는 플러그인이 아직 마켓플레이스에 없는 통합에 의존한다면, 설치는 누락된 의존성을 이름으로 표시하므로 다시 시도하기 전에 그것을 가져오거나(또는 게시할 수 있습니다) 할 수 있습니다.

## 연결이 있는 플러그인 설치

마켓플레이스 목록에서 **Install**을 클릭하면 Polymux는 각 필수 연결을 순서대로 안내하는 설치 대화상자를 엽니다.

1. **설명을 읽으세요.** 각 연결에는 레이블, 도움말 설명, 필수/선택 플래그가 있습니다.
2. **인증하세요.** OAuth 연결은 _Authorise_ 버튼을 표시합니다. vault 연결은 붙여넣기 필드를 표시합니다. 통합 연결은 관련 제공자 폼을 표시합니다.
3. **확인하세요.** 요약 화면은 플러그인이 접근하게 될 모든 것을 나열합니다. _Install_을 눌러 확정하세요.

플러그인이 워크스페이스의 플러그인 목록에 나타납니다. 플러그인이 자체 스케줄을 가지고 있지 않은 한 첫 실행은 수동으로 이루어집니다. 자체 스케줄이 있다면 Polymux는 스케줄에 따라 실행합니다.

## 끊어진 연결 재연결

플러그인 설정에서 끊어진 연결 옆의 **Reconnect**를 클릭하세요. 대화상자가 동일한 인증/붙여넣기/확인 흐름을 안내합니다.

OAuth 제공자의 경우 끊어진 연결의 가장 흔한 원인은 기본 계정이 삭제되었거나 제공자 측에서 OAuth 권한 부여가 취소되었기 때문입니다. Reconnect는 새 권한 부여를 깔끔하게 가져옵니다.

## 연결 제거

플러그인이 여전히 참조하는 동안 연결을 제거하면 플러그인이 저하된 상태로 들어갑니다. 선택적 연결은 조용히 제거됩니다. 필수 연결은 다음 실행에서 플러그인을 실패시킵니다.

**Vault → Connections**에서 연결을 제거할 수 있습니다. 제공자 측의 권한 부여는 자동으로 취소되지 않습니다. 이를 위해서는 제공자의 계정 설정으로 가서 거기서 취소하세요.
