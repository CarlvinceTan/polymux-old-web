# Vault 기본 사항

vault는 에이전트가 여러분을 대신해 작업하기 위해 필요한 자격 증명, API 키 및 기타 비밀 정보를 저장하는 곳입니다. 에이전트가 vault에서 읽는 모든 것은 로그로 기록되고 세션 범위로 한정되며, 모델에 그대로 반환되지 않습니다. 페이지나 도구 호출에 직접 주입됩니다.

## vault에 들어가는 것

vault 항목에는 두 가지 종류가 있습니다.

- **비밀번호** — 도메인으로 범위가 지정된 사용자 이름/비밀번호 쌍. 에이전트가 일치하는 URL에 도달하면 로그인 폼에 입력합니다.
- **지갑 항목** — API 키, bearer 토큰, OAuth 클라이언트 시크릿 및 기타 임의의 비밀 정보. 폼에 입력되는 대신 도구 호출(HTTP 요청 헤더, CLI 인수)에 주입됩니다.

vault는 파일, 인증서 또는 SSH 키를 저장하지 않습니다. 이러한 항목은 [workspace storage](/documentation/installation)를 사용하세요.

## 항목 추가

사이드 패널에서 **Vault → Passwords** 또는 **Vault → Wallet**을 열고 **+ New**를 누르세요. 항목에 이름, 호스트(비밀번호의 경우) 또는 키 형식(지갑 항목의 경우), 그리고 값을 지정하세요. 값은 워크스페이스 키로 저장 시 암호화됩니다. Polymux 엔지니어를 포함하여 누구도 여러분의 인증 없이는 다시 읽을 수 없습니다.

## 에이전트가 vault에 접근하는 방법

에이전트는 비밀 정보를 보지 못합니다. 워크플로가 비밀 정보를 필요로 할 때, _"github.com의 비밀번호"_ 같은 형식의 요청을 vault에 발행하고 Polymux는 에이전트의 컨텍스트 윈도우에 값을 넣지 않은 채 다음 동작에 그 값을 주입합니다. 모델은 비밀 정보가 존재하고 그것이 무엇을 위한 것인지 알지만, 실제 문자는 알지 못합니다.

세션이 로그인 폼 주변에서 일시 정지했다가 재개되는 것을 본다면, 그 일시 정지가 바로 vault가 자격 증명을 채우고 있는 순간입니다.

## 범위 지정

기본적으로 **Member** 이상의 역할을 가진 모든 워크스페이스 멤버는 워크플로에서 모든 vault 항목을 사용할 수 있습니다. 항목을 제한하려면 편집하고 범위를 설정하세요.

- **Workspace** — 워크스페이스 역할을 가진 누구나 사용할 수 있습니다. 기본값입니다.
- **Workflow** — 나열된 ID의 워크플로만 요청할 수 있습니다. 영향력이 큰 키에 유용합니다.
- **Owner only** — 항목의 생성자만 사용할 수 있으며, 예약된 실행에서도 마찬가지입니다.

아직 감사 로그 UI는 없지만, 모든 vault 읽기는 서버 측에서 캡처됩니다. 특정 항목의 로그를 가져와야 한다면 [contact support](/contact)에 문의하세요.

## 비밀 정보 교체

항목을 열고 **Rotate**를 클릭한 다음 새 값을 붙여넣으세요. 이전 값은 삭제됩니다. vault에는 버전 기록이 없습니다. 이전 값을 참조하던 워크플로는 다음 실행 시 계속 작동합니다. 진행 중인 세션은 종료될 때까지 메모리에 이전 값을 계속 유지합니다.

## 항목을 삭제하면 어떻게 되나요

삭제는 즉각적이며 복구할 수 없습니다. 삭제된 항목에 대한 vault 읽기에서 일시 정지된 세션은 `vault_missing`으로 실패하며 새 항목이 필요합니다. 예약된 실행도 같은 방식으로 실패합니다. 실패한 실행에 대한 알림이 대시보드에 표시됩니다.
