# 플러그인 개요

Polymux 플러그인은 다른 사람이 한 번의 클릭으로 자신의 워크스페이스에 설치할 수 있는 패키지화된 워크플로입니다. 플러그인은 워크플로 그래프, 필요한 연결(vault 키, OAuth 제공자, 통합), 그리고 목록을 설명하는 작은 매니페스트를 묶습니다.

이 페이지는 개발자 가이드의 출발점입니다. 아직 워크플로를 구축하지 않으셨다면 먼저 [Quickstart](/documentation/quickstart)를 따라 하세요.

## 워크플로를 플러그인으로 패키지화할 때

다음과 같은 경우 워크플로를 패키지화하세요.

- **다른 워크스페이스의 팀원**이 다시 작성하지 않고 사용할 수 있게 하고 싶을 때.
- 워크플로가 **모든 실행 사이에 편집하지 않을 만큼 안정적**일 때.
- **이식 가능한 연결**(OAuth 제공자, 공개 API 또는 설치자가 제공할 수 있는 vault 키)을 사용할 때.

워크플로가 워크스페이스 고유 데이터(프롬프트에 하드코딩된 멤버 이메일, 내부 호스트네임, 단일 테넌트 비밀 정보)를 포함한다면 패키지화하지 _마세요_. 플러그인을 설치하는 누구나 해당 데이터의 사본을 갖게 됩니다.

## 플러그인의 구성

모든 플러그인은 네 가지 부분으로 구성됩니다.

1. **워크플로 그래프.** 워크플로 편집기에서 작성한 노드, 엣지, 프롬프트, 도구 선택. 워크플로 자체의 버전 기록에 따라 버전이 관리됩니다.
2. **매니페스트.** 이름, 설명, 아이콘, 카테고리, 스크린샷, 가격. 마켓플레이스와 설치 대화상자에서 사용되는 표면 메타데이터입니다.
3. **연결 스키마.** 워크플로가 실행되기 위해 필요한 vault 키, OAuth 제공자, 통합 ID. Polymux는 이를 사용하여 설치 시 설치자에게 올바른 비밀 정보를 요청합니다.
4. **변경 로그.** 게시된 버전별 자유 형식의 릴리스 노트. 목록 페이지에 표시됩니다.

이어지는 페이지들은 각각을 차례로 안내합니다.

## 패키지화된 작업의 두 가지 형태

Polymux는 마켓플레이스에서 두 가지 관련 아티팩트를 지원합니다.

- **플러그인** — 패키지화된 워크플로. 워크스페이스로 가져와 설치자가 실행합니다.
- **연결** — 패키지화된 통합. 워크스페이스당 한 번 가져온 다음, 어떤 워크플로에서든 도구로 사용할 수 있습니다.

이 가이드는 대부분의 작성자가 처음 시작하는 것이기 때문에 플러그인에 초점을 맞춥니다. 연결은 [Building a connection](/documentation/connections-build)에 문서화되어 있습니다.

## 배포

플러그인은 세 가지 방식으로 게시할 수 있습니다.

- **공개 마켓플레이스** — [polymux.com/integrations/marketplace](/integrations/marketplace)에 등록됩니다. Polymux 계정을 가진 누구나 설치할 수 있습니다. 무료 또는 유료입니다.
- **워크스페이스 전용** — 단일 워크스페이스의 멤버에게만 표시됩니다. 공개하고 싶지 않은 내부 도구에 유용합니다.
- **비공개 링크** — 직접 URL로 접근할 수 있지만 색인화되지 않습니다. 비공개 베타나 Polymux 외부의 유료 배포에 유용합니다.

제출 시 배포를 선택합니다. 검토를 위해 다시 제출하지 않고도 나중에 변경할 수 있습니다.

## 가격 및 수익 공유

플러그인에 대해 일회성 또는 월간 요금을 부과할 수 있습니다. Polymux는 Stripe를 통해 결제를 처리하며 15%의 플랫폼 수수료를 받습니다. 나머지는 매월 여러분의 Stripe Connect 계정으로 지급됩니다.

무료 플러그인은 등록 수수료가 없고 결제 통합을 설정할 필요도 없습니다. 무료 릴리스로 시작한 다음 설치 수가 확보된 후 가격을 추가하는 것을 권장합니다.
