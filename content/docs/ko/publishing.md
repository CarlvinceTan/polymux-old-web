# 게시 체크리스트

플러그인을 구축했고, 목록을 작성했으며, Publish 탭의 _Review_ 패널이 올바르게 보입니다. **Submit for review**를 누르기 전에 이 체크리스트를 실행하세요. 대부분의 거절은 이 페이지의 무언가에 의해 발생합니다.

## 워크플로

- [ ] **cold start에서 실행됨.** 빈 채팅에서 새 실행을 트리거하세요. 워크플로는 여러분의 개입 없이 첫 턴에서 최종 상태에 도달해야 합니다.
- [ ] **하드코딩된 비밀 정보 없음.** 키, 토큰, 비밀번호 또는 이메일처럼 보이는 모든 것을 프롬프트에서 검색하세요. 모든 일치 항목을 vault로 옮기고 키로 참조하세요.
- [ ] **워크스페이스 고유 데이터 없음.** 내부 호스트네임, 멤버 이메일, 시트 ID, Slack 채널 — 설치자가 변경하고 싶을 수 있는 모든 것은 워크플로 입력이어야 합니다.
- [ ] **모든 도구가 첫 번째 당사자이거나 마켓플레이스에 게시됨.** 자신의 비공개 워크스페이스에서 도구를 가져오는 플러그인은 설치되지만 다른 사람에게는 절대 실행되지 않습니다.
- [ ] **실패 시 깔끔하게 중지됨.** 알려진 실패 경로(잘못된 vault 항목, 오프라인 제공자)를 트리거하고 워크플로가 영원히 회전하는 대신 유용한 오류를 표시하는지 확인하세요.

## 목록

- [ ] **이름**은 1–40자이며 버전 번호가 없습니다.
- [ ] **짧은 설명**은 한 문장이며 120자 미만이고, 플러그인이 _무엇으로 구축되었는지_가 아니라 _무엇을 하는지_를 말합니다.
- [ ] **긴 설명**은 플러그인이 입력으로 무엇을 필요로 하는지, 무엇을 생성하는지, 어떤 데이터를 다루는지 설명합니다. 검토자는 이것을 주의 깊게 읽습니다.
- [ ] **아이콘**은 정사각형이며 64×64에서 깔끔하게 렌더링됩니다.
- [ ] **스크린샷**은 마케팅 사이트가 아닌 사용 중인 플러그인을 보여줍니다. 2~4개가 적정 수준입니다.
- [ ] **카테고리**는 가장 가까운 일치 항목입니다. _Other_는 정말로 어울리지 않는 플러그인을 위한 것입니다.
- [ ] **태그**는 정확합니다. 관련 없는 용어로 채우지 마세요. 검색은 태그 채우기를 낮게 평가합니다.

## 연결

- [ ] **모든 필수 연결에 낯선 사람이 이해할 수 있는 `label`이 있음.** _"OpenAI API key"_는 좋고, _"oai_k"_는 좋지 않습니다.
- [ ] **모든 필수 연결에 `help` 설명이 있음.** 단, 레이블이 완전히 자명한 경우(OAuth 제공자는 보통 그렇습니다)는 예외입니다.
- [ ] **선택적 연결은 우아하게 저하됨.** 각 선택적 연결이 누락된 상태에서 실행을 트리거하고 워크플로가 해당 단계를 건너뛰거나 합리적인 오류를 반환하는지 확인하세요.
- [ ] **스코프는 최소화됨.** 워크플로가 작동하도록 하는 가장 좁은 OAuth 스코프를 요청하세요. 검토자는 광범위한 권한 부여에 대해 반대할 것입니다.

## 가격 (유료 플러그인만)

- [ ] **Stripe Connect가 설정됨.** `Settings → Payouts`에 인증된 계정이 표시되어야 합니다.
- [ ] **통화와 금액이 정확함.** 게시 후에는 지원에 문의하지 않고 목록의 통화를 변경할 수 없습니다.
- [ ] **환불 정책**이 긴 설명에 언급됨. 플랫폼 전체의 정책은 없으며 여러분이 설정합니다.

## 제출 후

- 검토는 영업일 기준 2일 이내에 완료됩니다. 결과가 담긴 이메일을 받게 됩니다.
- 거절되면 이메일에는 변경이 필요한 특정 필드가 나열됩니다. 수정하고 다시 제출하세요. 여러 제출에 대한 페널티는 없습니다.
- 승인되면 목록이 즉시 라이브가 됩니다. 마켓플레이스의 _New_ 탭이 처음 7일 동안 그것을 추천합니다.

## 출시 후 버저닝

워크플로에 푸시하는 모든 변경에 대해 버전을 올리세요.

- **Patch** (`1.0.0 → 1.0.1`) — 버그 수정, 동작 변경 없음, 새 연결 없음.
- **Minor** (`1.0.0 → 1.1.0`) — 새 선택적 동작, 선택적 연결, 기본값이 있는 추가 입력.
- **Major** (`1.0.0 → 2.0.0`) — 필수 새 연결, 입력 제거, 실질적으로 다른 동작.

설치자는 버전 범위에 고정할 수 있습니다. 메이저 증가는 업데이트 전에 재인증을 요청합니다.

## 게시 취소

Publish 탭에서 언제든지 마켓플레이스에서 플러그인을 내릴 수 있습니다. 기존 설치자는 그들이 제거할 때까지 플러그인이 계속 실행됩니다. 새 설치자는 그것을 찾을 수 없습니다.

플러그인을 **긴급 중단**해야 한다면 — 예를 들어 데이터 유출을 발견했다면 — 지원에 문의하세요. 모든 설치자에 대해 한 번에 서버 측에서 플러그인을 비활성화할 수 있습니다.
