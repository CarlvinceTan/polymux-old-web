# FAQ

주제별로 분류된 일반적인 질문들입니다. 질문이 여기에 없다면 [forum](/forum)이 가장 빠른 문의 경로입니다.

## 계정 및 결제

### 이메일 주소를 어떻게 변경하나요?

대시보드에서 **Settings → Account → Email**을 여세요. 변경이 적용되기 전에 이전 주소와 새 주소 모두에서 변경을 확인해야 합니다.

### 워크스페이스 간에 워크플로를 옮길 수 있나요?

직접적으로는 불가능합니다. 권장 경로는 워크플로를 [packaged workflow](/documentation/plugin-overview)로 게시하고 대상 워크스페이스에 설치하는 것입니다. 이렇게 하면 프롬프트, 도구, vault 참조는 가져가지만 실행 기록은 초기화됩니다.

### 취소하면 데이터는 어떻게 되나요?

무료 등급 데이터는 무기한 보관됩니다. 유료 플랜은 취소 시 무료 등급으로 다운그레이드되며, 이로 인해 사용량이 무료 할당량을 초과할 수 있습니다. Polymux는 할당량 이하로 되돌리기 위해 어떤 것을 삭제하기 전 30일 동안 앱 내에서 알림을 보냅니다.

## 세션 및 워크플로

### 세션이 왜 끊겼나요?

세션은 서버에서 계속 살아 있습니다. 로컬 브라우저 탭이 닫히거나 네트워크가 끊겨도 세션은 계속 실행되며 사이드 패널에서 다시 연결할 수 있습니다. 세션 자체가 종료되었다면 채팅 상단에 상태 코드가 표시됩니다. `idle_timeout`, `budget_exceeded` 또는 `error`가 일반적입니다.

### 세션은 얼마나 오래 실행할 수 있나요?

엄격한 실시간 한도는 없지만, 모든 세션은 워크플로에 설정된 토큰 예산을 갖습니다. 예산이 소진되면 세션이 일시 정지하고 계속하기 전에 승인을 요청합니다. 기본 예산은 대부분의 다중 시간 브라우저 작업을 처리할 만큼 넉넉합니다.

### 라이브 뷰포트가 왜 검은색인가요?

흔한 원인 세 가지입니다.

- **네트워크에서 WebRTC가 차단됨** — 뷰포트는 몇 초 후 더 느린 폴링 스트림으로 대체됩니다. 뷰포트 모서리의 연결 아이콘을 확인하세요. 노란색 점이 표시되면 대체 스트림 상태입니다.
- **에이전트가 아직 이동하지 않음.** 브라우저가 첫 페이지를 로드할 때까지 뷰포트는 비어 있습니다.
- **`?mode=extension`으로 실행 중**이지만 확장 프로그램이 페어링되지 않음. 확장 프로그램 팝업을 열고 상태 배지를 확인하세요.

### 권한 오류

워크플로가 권한 오류로 실행을 거부한다면, 가장 흔한 원인은 워크스페이스 역할이 워크플로 실행을 허용하지 않는 것입니다. Viewer는 실행을 시작할 수 없습니다. 관리자에게 **Member**로 업그레이드해 달라고 요청하세요.

## Vault 및 비밀 정보

### 모델이 내 비밀번호를 볼 수 있나요?

아닙니다. vault 값은 페이지 동작이나 도구 호출에 직접 주입됩니다. 모델은 비밀 정보가 사용되었다는 것을 들을 뿐 실제 문자를 보지는 못합니다. 전체 그림은 [Vault basics](/documentation/vault#how-agents-access-the-vault)를 참고하세요.

### 어떤 암호화를 사용하나요?

vault 항목은 워크스페이스별 키를 사용한 AES-256-GCM으로 저장 시 암호화됩니다. 워크스페이스 키 자체는 관리형 KMS에 보관된 루트 키로 래핑됩니다. 복호화된 값은 로그로 남기거나 저장하지 않습니다.

### vault 항목을 내보낼 수 있나요?

현재 일괄 내보내기는 없습니다. 각 항목은 vault 페이지에서 개별적으로 표시하고 복사할 수 있습니다. 내보내기 도구는 로드맵에 있으며, 추적 이슈는 [forum](/forum)에 있습니다.

## 플러그인 및 개발자 질문

### 플러그인이 무엇인가요?

_플러그인_은 패키지화된 워크플로와 그것의 연결을 합친 것입니다. 다른 사람이 한 번의 클릭으로 자신의 워크스페이스에 설치하는 데 필요한 모든 것입니다. [Plugin overview](/documentation/plugin-overview)를 참고하세요.

### 플러그인을 어떻게 게시하나요?

게시하려는 워크플로를 열고 **Publish** 탭으로 이동하여 목록 필드를 작성하고 검토를 위해 제출하세요. 검토는 보통 영업일 기준 2일 이내에 완료됩니다. 전체 안내는 [Publishing a plugin](/documentation/publishing)을 참고하세요.

### 플러그인이 서로의 데이터를 읽을 수 있나요?

아닙니다. 각 플러그인은 자체적인 vault 접근 범위를 가지고 자체 세션에서 실행됩니다. 같은 워크스페이스에 설치된 두 플러그인은 서로의 vault 읽기, 파일 또는 세션 기록을 볼 수 없습니다.

## 다운로드 검증

데스크톱 설치 프로그램과 확장 프로그램 `.zip` 아카이브는 서명되어 있습니다. 각 릴리스의 SHA-256은 다운로드 옆 [Install page](/install-apps)에 나열되어 있습니다. 예를 들어 macOS `.dmg`를 검증하려면 다음과 같이 합니다.

```sh
shasum -a 256 Polymux-1.0.0-universal.dmg
```

출력값을 설치 페이지의 값과 비교하세요. 일치하지 않으면 설치 프로그램을 실행하지 말고 [contact](/contact)를 통해 문의하세요.

## 여전히 막혔다면?

위의 어떤 내용도 질문에 답하지 못한다면 다음을 시도하세요.

1. [forum](/forum)에서 검색하세요. 일반적인 문제는 보통 거기서 논의됩니다.
2. 프로그래밍 방식으로 통합 중이라면 [API overview](/documentation/api-overview)를 훑어보세요.
3. [contact page](/contact)에서 지원팀에 이메일을 보내세요. 워크스페이스 ID와 세션 ID(URL에 표시됨)를 포함하면 로그에서 여러분의 실행을 찾을 수 있습니다.
