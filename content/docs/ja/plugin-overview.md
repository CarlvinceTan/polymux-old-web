# プラグイン概要

Polymuxプラグインは、他の人が自分のワークスペースに1クリックでインストールできるパッケージ化されたワークフローです。プラグインには、ワークフローグラフ、必要な接続(vaultキー、OAuthプロバイダー、インテグレーション)、およびリスティングを記述する小さなマニフェストがバンドルされています。

このページは開発者ガイドの出発点です。まだワークフローを構築していない場合は、先に[Quickstart](/documentation/quickstart)に従ってください。

## ワークフローをプラグインとしてパッケージ化するタイミング

次のような場合にワークフローをパッケージ化しましょう。

- **他のワークスペースのチームメイト** が再作成せずに使えるようにしたい場合。
- ワークフローが **十分に安定していて、実行ごとに編集しない** 場合。
- **ポータブルな接続** を使う場合。OAuthプロバイダー、パブリックAPI、またはインストーラーが提供できるvaultキーなど。

ワークフローにワークスペース固有のデータ(プロンプトにハードコードされたメンバーのメール、内部ホスト名、シングルテナントの機密情報)が埋め込まれている場合は、パッケージ化 _しない_ でください。プラグインをインストールした人は、そのデータのコピーを取得することになります。

## プラグインの構造

すべてのプラグインは4つの要素で構成されています。

1. **ワークフローグラフ。** ワークフローエディタで作成したノード、エッジ、プロンプト、ツール選択。ワークフロー自身のバージョン履歴ごとにバージョン管理されます。
2. **マニフェスト。** 名前、説明、アイコン、カテゴリ、スクリーンショット、価格。マーケットプレイスとインストールダイアログで使用される表面メタデータ。
3. **接続スキーマ。** ワークフローが動作するために必要なvaultキー、OAuthプロバイダー、インテグレーションID。Polymuxはこれを使って、インストール時にインストーラーから適切な機密情報を求めます。
4. **変更履歴。** 公開バージョンごとのフリーフォームのリリースノート。リスティングページに表示されます。

次のページではこれらをそれぞれ詳しく説明します。

## パッケージ化された作業の2つの種類

Polymuxはマーケットプレイスで関連する2つのアーティファクトをサポートしています。

- **Plugins** — パッケージ化されたワークフロー。ワークスペースにインポートされ、インストーラーによって実行されます。
- **Connections** — パッケージ化されたインテグレーション。ワークスペースごとに1回インポートされ、任意のワークフローでツールとして利用可能になります。

このガイドはプラグインに焦点を当てています。ほとんどの作者がそこから始めるからです。接続については[Building a connection](/documentation/connections-build)で説明します。

## 配布

プラグインは3つの方法で公開できます。

- **公開マーケットプレイス** — [polymux.com/integrations/marketplace](/integrations/marketplace)にリスティングされます。Polymuxアカウントを持つ誰でもインストールできます。無料または有料です。
- **ワークスペース限定** — 単一のワークスペースのメンバーにのみ表示されます。公開したくない内部ツールに有用です。
- **非公開リンク** — 直接URLからアクセスできますがインデックスされません。クローズドベータやPolymux外での有料配布に有用です。

提出時に配布方法を選択します。後で再レビューなしに変更できます。

## 価格設定とレベニューシェア

プラグインには一回払いまたは月額料金を設定できます。PolymuxはStripeを通して支払いを処理し、15%のプラットフォーム手数料を取ります。残額は毎月、Stripe Connectアカウントに支払われます。

無料プラグインにはリスティング手数料や決済統合のセットアップは不要です。まずは無料リリースから始め、インストール数が増えてから価格を追加することをおすすめします。
