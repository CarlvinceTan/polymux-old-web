# 公開チェックリスト

プラグインを構築し、リスティングを入力し、Publishタブの _Review_ パネルが正しく見えています。**Submit for review** を押す前に、このチェックリストを実行してください。却下のほとんどはこのページ上の項目が原因です。

## ワークフロー

- [ ] **コールドスタートで実行できる。** 空のチャットで新しい実行をトリガーする。ワークフローは、あなたが介入することなく、最初のターンで終端状態に到達しなければなりません。
- [ ] **ハードコードされた機密情報がない。** プロンプトでキー、トークン、パスワード、メールに見えるものを検索してください。一致したものをすべてvaultに移動し、キーで参照してください。
- [ ] **ワークスペース固有のデータがない。** 内部ホスト名、メンバーのメール、シートID、Slackチャンネル。インストーラーが変更したい可能性のあるものはすべて、ワークフロー入力にしなければなりません。
- [ ] **すべてのツールがファーストパーティまたはマーケットプレイス公開済み。** あなたのプライベートワークスペースからツールをインポートするプラグインは、インストールはできても他の人には決して実行されません。
- [ ] **失敗時に正常に停止する。** 既知の失敗パス(間違ったvaultエントリ、オフラインプロバイダー)をトリガーし、ワークフローが無限に回転するのではなく有用なエラーを表示することを確認してください。

## リスティング

- [ ] **Name** は1〜40文字、バージョン番号なし。
- [ ] **Short description** は1文、120文字未満、プラグインが _何をするか_(_何で構築されているか_ ではなく)を述べる。
- [ ] **Long description** はプラグインが入力として必要とするもの、生成するもの、触れるデータを説明する。レビュアーはこれを注意深く読みます。
- [ ] **Icon** は正方形で、64×64でクリーンにレンダリングされる。
- [ ] **Screenshots** はマーケティングサイトではなく、プラグインの使用中の様子を示す。2〜4枚が最適。
- [ ] **Category** は最も近いマッチ。_Other_ は本当に当てはまらないプラグイン用です。
- [ ] **Tags** は正確。無関係な用語で水増ししないでください。検索はタグ詰めを低くランク付けします。

## 接続

- [ ] **必須の接続にはすべて、見知らぬ人が理解できる`label`がある。** _"OpenAI API key"_ は良いですが、_"oai_k"_ はダメです。
- [ ] **必須の接続にはすべて`help`の説明がある。** ただしラベルが完全に自明な場合(OAuthプロバイダーは通常自明)は不要です。
- [ ] **オプションの接続は優雅に縮退する。** 各オプション接続が欠けた状態で実行をトリガーし、ワークフローがそのステップをスキップするか、合理的なエラーを返すことを確認してください。
- [ ] **スコープは最小限。** ワークフローが機能する最小のOAuthスコープを要求してください。レビュアーは広範な権限付与に反対します。

## 価格設定(有料プラグインのみ)

- [ ] **Stripe Connectが設定済み。** `Settings → Payouts`に検証済みアカウントが表示されているはず。
- [ ] **通貨と金額が正しい。** 公開後にサポートに連絡せずに通貨を変更することはできません。
- [ ] **返金ポリシー** が長い説明に記載されている。プラットフォーム全体のポリシーはありません。あなたが設定してください。

## 提出後

- レビューは2営業日以内に完了します。結果はメールで届きます。
- 却下された場合、メールには変更が必要な特定のフィールドが記載されます。修正して再提出してください。複数回の提出にペナルティはありません。
- 承認されると、リスティングはすぐに公開されます。マーケットプレイスの _New_ タブで最初の7日間紹介されます。

## ローンチ後のバージョニング

ワークフローへの変更ごとにバージョンを上げてください。

- **Patch** (`1.0.0 → 1.0.1`) — バグ修正、動作変更なし、新しい接続なし。
- **Minor** (`1.0.0 → 1.1.0`) — 新しいオプション動作、オプション接続、デフォルト値を持つ追加入力。
- **Major** (`1.0.0 → 2.0.0`) — 必須の新しい接続、入力の削除、動作の大幅な変更。

インストーラーはバージョン範囲を固定できます。メジャーバンプの際は、アップデート前に再認可を促されます。

## 公開解除

PublishタブからいつでもプラグインをマーケットプレイスからPullできます。既存のインストーラーは、アンインストールするまでプラグインを実行し続けます。新規インストーラーはそれを見つけることができません。

プラグインを **緊急停止** する必要がある場合(たとえば、データ漏洩を発見した場合)は、サポートに連絡してください。サーバー側で全インストーラーに対して一度にプラグインを無効化できます。
