# 最初のプラグインを構築する

このウォークスルーでは、既存のワークフローを取り上げ、インストール可能なプラグインとしてパッケージ化します。[Plugin overview](/documentation/plugin-overview)を読み、エンドツーエンドでクリーンに実行されるワークフローがワークスペースにあることを前提としています。

## 1. 候補のワークフローを選ぶ

公開したいワークフローを開いてください。何かする前に、確認作業を行いましょう。

- **コールドスタートで実行できますか?** 空のチャットで新しい実行をトリガーしてください。事前のコンテキスト(以前にアップロードされたファイル、セッション中に作成されたvaultエントリ)に依存するものは、ゼロから始めるインストーラーでは失敗します。
- **すべての機密情報はvaultに入っていますか?** 機密情報がプロンプトにそのまま貼り付けられている場合、プラグインをインストールした人もそれを取得します。vaultに移動してキーで参照してください。
- **URLとIDは設定可能ですか?** ハードコードされた組織名、シートID、ホスト名を、インストーラーが入力できるワークフロー入力に置き換えてください。

このチェックリストをパスしたワークフローは、パッケージ化に適した状態です。

## 2. Publishタブを開く

ワークフローページで **Publish** タブに切り替えます。表示されない場合、ワークスペースのロールに公開権限が含まれていません。管理者に **Member** 以上へのアップグレードを依頼してください。

Publishタブには4つのサブパネルがあります。**Listing**、**Connections**、**Pricing**、**Review**。これらを順番に入力していきます。

## 3. Listing

リスティングは、マーケットプレイスカードと詳細ページで購入者が見る情報です。

- **Name.** 1〜40文字。末尾のバージョン番号は避けてください。バージョンは別途追跡されます。
- **Short description.** 1文、120文字以内。カードに表示されます。
- **Long description.** マークダウン対応。ワークフローが何をするか、何が必要か、何を生成するかを説明してください。スクリーンショットも有用です。
- **Icon.** 正方形のPNG、最小256×256。SVGも可。
- **Category.** 最も近いものを選んでください。カテゴリはマーケットプレイスのフィルタリングを駆動します。
- **Tags.** 最大5個のフリーフォームタグ。検索に使用されます。

## 4. Connections

Polymuxはワークフローグラフをスキャンし、見つかったすべての外部依存関係を提示します。vaultキー、OAuthプロバイダー、インテグレーションID。それぞれについて、次のことを宣言します。

- **必須かオプションか。** オプションの接続は、提供されない場合にワークフローが縮退モードで実行されることを許可します。
- **表示ラベル.** インストーラーがインストールダイアログで見る内容。生のvaultキー名よりも _"OpenAI API key"_ の方が適切です。
- **ヘルプテキスト.** インストーラーがどこで値を取得すべきかを説明する短いヒント。関連する場合はプロバイダーのドキュメントへのリンクを含めてください。

接続がビルトインのGoogle Drive OAuthプロバイダーの場合、インストーラーはインストール中にワンクリックで認可します。それ以外のプロバイダーでは、ワークフローはマーケットプレイスインテグレーション(またはインストーラーが貼り付けるvaultキー)に依存します。どちらも同じインストールダイアログで処理されます。

ここで何がキャプチャされるかの完全なスキーマについては[Plugin manifest reference](/documentation/plugin-manifest)を参照してください。

## 5. Pricing

3つのオプション:

- **Free.** 支払いなし、Stripeセットアップ不要。
- **One-time.** インストーラーが一度支払い、ワークスペースで無期限にプラグインを所有します。
- **Subscription.** 月額継続課金。インストーラーはいつでもキャンセルでき、期間終了時にプラグインがアンインストールされます。

有料プラグインの場合、**Settings → Payouts** からStripeアカウントを接続する必要があります。Polymuxは15%のプラットフォーム手数料を保持し、残額を毎月支払います。最低支払い閾値は$50です。

## 6. レビューと提出

**Review** サブパネルは、インストーラーが見るのと同じリスティングのプレビューを表示します。スクリーンショットを確認し、接続プロンプトをクリックして進み、長い説明にタイポがないか読んでください。

**Submit for review** を押してください。プラグインがキューに入り、2営業日以内に結果がメールで届きます。よくある却下理由は次のとおりです。

- リスティングの説明が、プラグインが実際に何をするか、どのデータに触れるかを省略している。
- 必須の接続が、見知らぬ人が理解できるほど明確にラベル付けされていない。
- ワークフローに、vaultに格納すべき機密情報が埋め込まれている。

リスティングを編集して、何度でも再提出できます。

## 7. アップデートの公開

プラグインが公開されると、ベースとなるワークフローへの変更は _新しいバージョン_ になります。バージョンは自動公開されません。Publishタブから **Promote to public** を選択して、既存のインストーラーがバージョンを利用できるようにしてください。彼らはアップデートするか、古いバージョンに留まるかを選べます。

破壊的な変更(新しい接続が必要になるなど)は、メジャーバージョンをインクリメントしなければなりません。Polymuxは、アップデートを適用する前に新しい権限を求められることをインストーラーに警告します。
