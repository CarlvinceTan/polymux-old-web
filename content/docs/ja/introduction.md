# はじめに

Polymuxは、あなたに代わってウェブを操作するAIエージェントを構築、実行、共有するためのプラットフォームです。エージェントは実際のブラウザセッションを動かし、互いに引き継ぎ、機密情報をvaultに保存し、実行を再生します。すべてはチームが管理するワークスペースから行えます。

このドキュメントは2部構成になっています。**ユーザーガイド**では、Polymuxのインストール、セットアップ、日々の利用方法を説明します。**開発者ガイド**では、パッケージ化されたワークフローや接続を構築・公開し、他の人がワンクリックでインストールできるようにする方法を解説します。

Polymuxを初めて利用する場合は、[Quickstart](/documentation/quickstart)から始めてください。APIリファレンスをお探しの場合は、[API overview](/documentation/api-overview)に直接お進みください。

## Polymuxが提供するもの

Polymuxは3つの基本要素を中心に構築されています。

- **Sessions** — ライブブラウザを背後に持つ1つのエージェントとの会話。セッションはすべてのクリック、スクロール、フォーム入力をダッシュボードにストリーミングします。
- **Workflows** — エージェント、ビューポート、ツールの有向グラフ。ワークフローはバージョン管理され、スケジュール実行可能で、マーケットプレイス向けにパッケージ化できます。
- **Workspaces** — チームのワークフロー、vaultエントリ、ファイル、請求情報を格納する共有コンテナ。メンバーは招待によって追加され、役割が割り当てられます。

これら3つの概念は、ドキュメントの他のすべての箇所で繰り返し登場します。あるセクションでこれらを説明なしに使っている場合は、このページに戻って確認してください。

## このドキュメントの読み方

左サイドバーはページをセクション別にグループ化しています。右側のレールは現在のページ上での位置を追跡し、見出し間のジャンプを可能にします。すべてのページの末尾には、読み順における前後のページへのリンクがあり、サイドバーに戻ることなくガイドを最初から最後まで通読できます。

Polymuxは活発に開発中です。製品が変更されるたびにこれらのページを更新していますので、古いガイドをお読みの場合は再確認することをおすすめします。
